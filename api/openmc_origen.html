OpenMC+Origen – xsgen.openmc_origen¶

An engine that uses OpenMC and ORIGEN2.2 to do burnup-criticality computations.

class xsgen.openmc_origen.OpenMCOrigen(rc)¶

An that combines OpenMC for k-code calculations and ORIGEN for transmutation.

context(state)¶

Unite parameters in the run-control file and the current state.

state : namedtuple (State)
A State tuple that contains perturbation parameters specific to that state.
ctx : dict
A dictionary with all relevant perturbation parameters.
generate(state, transmute_time)¶

Runs physics codes on a specific state. First runs OpenMC for transport, then uses the results to run ORIGEN for a single timestep.

state : namedtuple (State)
A namedtuple containing the state parameters.
transmute_time : float
The length of the time step we would like to run ORIGEN for.
results : dict
Dict of physics code results. Keys are either nuclide ID’s or “fuel” for the full fuel results.
generate_run(run)¶

Generate transmutation tables, neutron production/destruction rates, and burnup statistics for a sequence of states with the same initial conditions.

run : list of States
A list of States that has the same initial conditions at increasing burnup times.
libs : list of dicts
Libraries to write out - one for the full fuel and one for each tracked nuclide.
nucs_in_cross_sections()¶

Returns the set of nuclides present in the cross_sections.xml file.

nucs : set
A set of all nuclides known to OpenMC, in ID form.
